1   Overview

1.1   What is NEXIM?

NEXIM (Novel End-to-eXtended hyperspectral IMage simulator) is a free, open-source, physics-based hyperspectral scene simulator for Windows. It generates synthetic radiance cubes in the UV–Far-IR spectral range (0.4–2.5 µm and beyond), comparable in accuracy to CameoSim, DIRSIG, and MODTRAN-based tools — with zero commercial dependencies.

NEXIM combines a native C# correlated-k distribution (CKD) atmospheric radiative-transfer engine with a three-mode accuracy/speed tradeoff, an optional libRadtran subprocess for maximum fidelity, a spectral material library pipeline driven by real RGB imagery, and eight segmentation algorithms — all packaged in a single Windows Forms desktop application.

1.2   Three Atmospheric RT Modes

ModeEngineAccuracySpeedExternal dep.
1 FAST Pre-computed LUT (~50 MB bundled) ±5–10 % < 10 ms None
2 ACCURATE Native C# CKD + DISORT 8-stream ±0.5–1 % (MODTRAN-class) 100–500 ms None
3 FULL-PHYSICS libRadtran via WSL2 subprocess ±0.1–0.5 % 5–30 s WSL2 + libRadtran

1.3   Novel Contributions

  • First MODTRAN-class atmospheric RT engine in pure C#, freely redistributable.
  • First zero-dependency broadband (UV–Far-IR) scene simulator on Windows desktop.
  • Image-driven per-pixel material pipeline: load an RGB photo, cluster it, and assign spectra automatically via CIE-Lab nearest-neighbour matching.
  • Eight segmentation algorithms (K-means, GMM, DBSCAN, SAM, …) with a unified comparison dashboard.
  • Unified reflective + emissive pipeline (UV–Far-IR) in one tool.
  • Three-mode accuracy/speed tradeoff with quantified error per mode.

Install Steps

  1. Download NEXIM_Setup_v1.0.exe from the GitHub Releases page.
  2. Run the installer (may prompt for .NET 9 Desktop Runtime download — allow it).
  3. Launch NEXIM from the Start Menu or desktop shortcut.
  4. Optional: For Mode 3, open Tools → Install libRadtran and follow the wizard. WSL2 must be enabled in Windows Features first.
Note The application data directory is %APPDATA%\NEXIM. Spectral library downloads and LUT files are stored here and persist across updates.

3   Image Input Tab

The Image Input tab (first tab) drives the image-based simulation pipeline: load an RGB photograph of your scene, cluster its pixels into material classes, match each class to a spectral signature from the library, and create a per-pixel MaterialMap that the simulator uses to assign realistic reflectance spectra to every pixel.

If no image is loaded, the simulation falls back to a flat-reflectance scene (controlled by the Albedo slider on the Scene Setup tab).

[Screenshot: Image Input tab with source image and cluster map side by side]

3.1   Loading a Source Image

The left panel of the top row shows the Source Image preview.

  1. Click Browse… next to the path box.
  2. Select any BMP, PNG, JPEG, GIF, or TIFF file. Colour depth can be 8-bit palette, 24-bit RGB, or 32-bit ARGB; NEXIM converts internally to 32-bit ARGB.
  3. A thumbnail appears in the preview box. The status bar shows the pixel dimensions.
Tip Images from any nadir or oblique camera work well. Aerial/satellite imagery is ideal. Very large images (>4000 × 4000 px) are supported but classification will be slower.

3.2   Unsupervised Classification

Choose a clustering method, set its parameters, and click ▶ Classify Image. The cluster map appears on the right panel of the top row — each colour represents one material cluster.

Methods

MethodParameterBest for
K-means Clusters — number of classes (2–64, default 6) Fast, reliable for most scenes; good starting point
GMM Clusters — same as K-means Soft assignments; better when cluster sizes vary greatly
DBSCAN ε (0.01–1.0) and Min points (1–200) Automatically discovers number of clusters; handles irregular shapes; noise pixels (label −1) are mapped to cluster 0
Note Clustering is performed in RGB feature space (R/255, G/255, B/255 ∈ [0,1]). For ε in DBSCAN, a value of 0.10 means two pixels must be within 10 % of full colour range to be in the same cluster.

3.3   Spectral Library

After clustering, each cluster centroid's RGB colour is matched to the nearest spectral signature in the selected library using CIE-Lab ΔE76 colour distance. The match is shown in the Material Assignments grid.

Library Sources

SourceSpectraSetup required
Embedded 23 curated spectra (vegetation, soils, urban, water, minerals, snow) None — always available
USGS splib07b curated ~26 hand-picked spectra Click ⬇ Download Curated. Internet connection required once.
USGS splib07b full Hundreds per chapter (8 chapters available) Click ⬇ Download Full USGS Library…, select chapters. Chapter 7 (Vegetation) has 100+ spectra alone.
Custom CSV Unlimited — user-supplied Browse to a CSV file (see §10 for format)
Tip: Downloading the full USGS library Select all 8 chapters for a comprehensive library covering minerals, soils, vegetation, water, urban materials, coatings, organics, and mixtures. Downloads are cached; re-running the download only fetches new files. Expect 50–300 MB per chapter depending on chapter size.

3.4   Material Assignments

The Material Assignments grid shows, for each cluster:

  • # — cluster number (1-based)
  • Centroid — the mean RGB colour of the cluster (hex + colour swatch)
  • Matched Spectrum — the closest spectral signature in the library
  • Category — material category (Vegetation, Soil, Urban, etc.)

Click ↺ Refresh Assignments to re-run the CIE-Lab matching after changing the spectral library selection without re-running the clustering.

7   Segmentation Tab

The Segmentation tab runs spectral segmentation on the simulated radiance cube after simulation completes, and displays a false-colour label map.

Unlike the Image Input tab (which clusters RGB pixels for material assignment), this tab clusters the full-spectrum radiance vectors — providing a spectral classification of the simulated scene.

AlgorithmParametersNotes
K-meansClustersLloyd's algorithm; pure C# implementation
GMMClustersEM with diagonal covariance; initialised by K-means
DBSCANε, Min pointsDensity-based; discovers cluster count automatically
SAMThreshold (°), EndmembersSpectral Angle Mapper; add endmembers with Add endmember…

SAM Endmembers

Click Add endmember… to define a reference spectrum by name and comma-separated reflectance values. If no endmembers are added, NEXIM auto-derives them from K-means centroids.

8   Running a Simulation

  1. (Optional) Load an RGB image on the Image Input tab, classify it, and verify the material assignments.
  2. Configure scene geometry on the Scene Setup tab. If an image is loaded, Rows and Columns are automatically taken from the image dimensions.
  3. Select the atmospheric RT mode and profile on the Atmosphere tab. For quick test runs, use FAST or ACCURATE.
  4. Configure the sensor on the Sensor tab. For a default hyperspectral sensor, the factory settings (32 bands, Gaussian SRF) are a good starting point.
  5. Click ▶ Run Simulation at the bottom of the window. Progress is shown in the status bar and progress bar.
  6. When complete, the status bar shows the scene dimensions, band count, and elapsed time. If segmentation is enabled on the Segmentation tab, the label map is rendered automatically.
Performance tip For large images (e.g., 1000 × 1000 px), the image-driven pipeline renders each pixel individually. Use Mode 1 FAST for initial exploration; switch to Mode 2 ACCURATE for final output. Mode 2 runtime scales linearly with pixel count.
Image-driven simulation When a MaterialMap is active, each pixel's reflectance is taken from the matched spectral signature. The radiance model is: 
L(λ) = L_path(λ)  +  T(λ) · ρ_pixel(λ) · E_d(λ) / π

where L_path is atmospheric path radiance, T is transmittance, ρ_pixel is the per-pixel spectral reflectance, and E_d is downwelling solar irradiance at the surface.

11   Technical Reference

Per-pixel Radiance Model

For image-driven simulations, the at-sensor spectral radiance for pixel (r,c) is:

L(r,c,λ) = L_path(λ)  +  T(λ) · ρ(r,c,λ) · E_d(λ) / π

  L_path(λ)   — atmospheric path radiance [W m⁻² sr⁻¹ µm⁻¹]
  T(λ)        — total atmospheric transmittance (0–1)
  ρ(r,c,λ)   — per-pixel spectral reflectance from MaterialMap
  E_d(λ)      — downwelling solar irradiance at surface [W m⁻² µm⁻¹]

CIE-Lab Material Matching

Each cluster centroid's RGB colour is converted to CIE XYZ using the sRGB primaries, then to CIE L*a*b* using the D65 illuminant (X_n = 95.047, Y_n = 100, Z_n = 108.883). Each spectral signature is also converted to CIE Lab using a 43-point colour-matching function (CMF) evaluated at the embedded wavelength grid. The best match minimises ΔE₇₆ (Euclidean distance in Lab space).

Output File Formats

NXI format

[Header block]  ASCII JSON, UTF-8, NUL-terminated, padded to 4096 bytes
[Data block]    float32 BIL (Band-Interleaved-by-Line), rows × bands × cols
[Footer]        4-byte CRC32 of the data block (IEEE 802.3)

12   Troubleshooting

SymptomLikely causeFix
Run fails with "LUT not found" Mode 1 FAST selected but LUT file absent Switch to Mode 2 ACCURATE, or download/generate the LUT
Cluster map shows blank/black image Image not fully classified yet Click ▶ Classify Image first; wait for "N cluster(s) found" status
USGS download returns 0 spectra Server unavailable or URL changed Check internet connection; try the curated download first; use Embedded as fallback
Mode 3 fails with "WSL2 not found" WSL2 not installed or libRadtran not compiled Use Tools → Install libRadtran wizard; ensure WSL2 is enabled in Windows Features
Simulation very slow (> 30 s) Large image + Mode 2 ACCURATE Downscale the source image or switch to Mode 1 FAST for prototyping
Export fails with path error Destination directory does not exist Create the output directory before exporting
Labels appear truncated in panels Form resized below minimum width Increase the form width; minimum is 880 × 580 px; hover over truncated labels to see full text
